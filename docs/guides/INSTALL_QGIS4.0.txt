 OPTION 1 : INSTALLATION DIRECTE (Recommandé dev)
───────────────────────────────────────────────────────────────────────────────

1. TROUVER LE DOSSIER PLUGINS QGIS

   Chemin standard Windows :
   C:\Users\<YourUsername>\AppData\Roaming\QGIS\QGIS3\profiles\default\python\plugins

   Ou dans QGIS : Settings → User Profiles → Open Profile Folder
   → puis : python\plugins


2. COPIER LE PLUGIN

   Option A : Copie directe (simple)
   ─────────────────────────────────
   - Copier le dossier :
     c:\Users\camil\Desktop\...\QGISIA\QGISIA2

   - Vers :
     C:\Users\<YourUsername>\AppData\Roaming\QGIS\QGIS3\profiles\default\python\plugins\QGISIA2

   Option B : Symlink (meilleur pour dev)
   ──────────────────────────────────────
   PowerShell (Admin) :
   ```
   New-Item -ItemType SymbolicLink `
     -Path "C:\Users\$env:USERNAME\AppData\Roaming\QGIS\QGIS3\profiles\default\python\plugins\QGISIA2" `
     -Target "c:\Users\camil\Desktop\Micro Entreprise\04_PROJETS_EN_COURS\Projet\QGISIA\QGISIA2"
   ```

   Avantage : modifications en live sans recopy


3. REDÉMARRER QGIS

   - Fermer QGIS complètement
   - Rouvrir QGIS 4.0
   - Plugin → QGISAI+ doit apparaître dans la toolbar

 DÉPANNAGE
───────────────────────────────────────────────────────────────────────────────

❌ Plugin n'apparaît pas après redémarrage
   → Vérifier le chemin plugins (Settings → User Profiles)
   → Logs : Help → View → Python Console (rechercher erreurs)
   → Réinstaller en copie simple (pas symlink)

❌ "Erreur de port déjà utilisé"
   → Env var QGIS_BRIDGE_TIMEOUT → change port dynamique
   → Ou relancer QGIS

❌ Modal n'apparaît pas
   → F12 (DevTools) → Network tab
   → Chercher /api/llm/agent dans Network
   → Vérifier status 200

❌ Agent s'arrête après ask_user
   → Bug fixé (commit 69a17db)
   → Si persiste: Help → View → Log Messages (chercher erreurs Python)
